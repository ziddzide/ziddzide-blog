# IP地址

```
ip地址 = [网络位] + [主机位] (32位二进制)
```

## 子网掩码

用于区分网络位和主机位,一般ip地址和子网掩码复合表示为

```
192.168.1.2/30
```

## 广播地址和网络地址

### 广播地址

```
用于同一网段下的广播信息呼叫,例如arp请求mac地址,该地址将会转发给其他主机
```

### 网络地址

```
一般是网关或者路由器的地址,负责同网段下的设备流量进出网,阻止广播风暴
```
## 额外的内容

### 交换机,家用/企业路由器和广播地址

#### 交换机的核心作用

让连接在交换机上的设备可以互相发送数据,是广播地址的核心处理机器,负责把网段的广播信息转发给其他设备,同时也负责相互通信

#### 企业路由器不能处理广播地址

企业路由器的核心作用是阻止广播风暴隔离网段,一般网关的路由地址是网络地址,面对广播信息如果不是和自己相关将会被丢弃

#### 家用路由器为什么可以处理广播地址

家用路由器核心其实是三层交换机,可以处理第二层的信息交换,也可以处理第三层的路由,因此同时具备路由器和交换机的功能

### 网络地址和网段

#### 子网掩码相同的情况下一定属于同一内网环境吗?

```
192.168.1.0/30的情况下,网络地址是192.168.1.0,广播地址是192.168.1.3,中间有两台主机
192.168.1.4/30的情况下,网络地址是192.168.1.4,广播地址是192.168.1.7,中间有两台主机
所以,192.168.1.2/30和192.168.1.6/30并不属于同一网段,无法通信,只能让网络地址转发流量
```

#### 一些特殊的情况

```
 路由器A ──────────────── 路由器B
        Gig0/0            │
        192.168.1.1/30    │
                          │
                        路由器C
                        Gig0/0 = 192.168.1.6/30
                        广播地址 = 192.168.1.7
                        两个路由器A用它自己的192.168.1.5/30和C 192.168.1.6/30连接
```
