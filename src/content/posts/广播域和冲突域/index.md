## 冲突域

### 定义

冲突域指的是网络中一个设备发送数据时，其他设备必须等待，不能同时发送的区域。如果两台设备同时发，就会发生**冲突**，数据双双损坏，需要重传。

### 设备对冲突域的影响

| 设备 | 冲突域数量 |
|------|-----------|
| **集线器 Hub** | 所有端口在一个冲突域（1个） |
| **交换机 Switch** | 每个端口是一个独立冲突域（N个） |
| **路由器 Router** | 每个端口是一个独立冲突域（N个） |

### 集线器为什么是一个冲突域

集线器本质上是**物理层**设备，它不识别数据内容，只是把收到的信号复制到所有端口。所以 A 发数据给 B 时，C 和 D 的端口上也有信号，C 和 D 就不能发。

### 交换机为什么是多个冲突域

交换机是**数据链路层**设备，它维护 MAC 地址表。A 发数据给 B 时，交换机只把数据转发到 B 的端口，C 和 D 的端口上没有信号，它们可以同时发。所以每个端口独立。

> 这就是**交换机比集线器效率高**的根本原因：可以多对设备同时通信而不冲突。

### 冲突域的范围

- 介质为**同轴电缆**或**双绞线**的总线型结构：整个网段是一个冲突域
- 通过**集线器**组网：整个集线器下所有设备在一个冲突域
- 通过**交换机**组网：每个端口单独一个冲突域
- 通过**网桥**：分割冲突域（但广播域不变）

## 广播域

### 定义

广播域指的是网络中一个设备发送**广播帧**（目的 MAC 全 F）时，能收到这个广播的所有设备的集合。

### 设备对广播域的影响

| 设备 | 广播域数量 |
|------|-----------|
| **集线器 Hub** | 1个（所有端口转发广播） |
| **交换机 Switch** | 1个（默认转发广播到所有端口） |
| **路由器 Router** | **N个（不转发广播，分割广播域）** |

### 交换机的广播处理

交换机收到广播帧（目的 MAC = FF-FF-FF-FF-FF-FF）时，会从**除了接收端口外的所有端口**转发出去。所以同一台交换机下的所有设备在一个广播域内。

### 路由器为什么分割广播域

路由器是**网络层**设备，它的设计原则是"不转发广播"。广播只在本地网段内传播，路由器隔离了不同网段之间的广播流量。

> 这就是为什么不同 VLAN 之间通信需要三层路由——交换机隔离不了广播域，必须用路由器/三层交换。

## 冲突域 vs 广播域 对比

| 对比维度 | 冲突域 | 广播域 |
|---------|--------|--------|
| 关注对象 | 数据是否**同时发送** | 广播能否**传过去** |
| 分割设备 | **交换机/网桥** | **路由器/VLAN** |
| OSI 层次 | 主要影响第1、2层 | 主要影响第2、3层 |
| 影响性能的方式 | 冲突增多 → 重传 → 性能下降 | 广播泛滥 → 占用带宽 → 影响所有设备 |
| 典型故障表现 | 网络卡顿、丢包严重 | 全网变慢，抓包全是广播 |

## 网络优化思路

1. **用交换机替代集线器** → 缩小冲突域（从全局变为每端口）
2. **划分 VLAN** → 缩小广播域
3. **端口隔离** → 特定端口不允许广播互达
4. **风暴控制** → 限制广播包速率，防止广播风暴
