# Linux网卡配置

> 以 ubuntu为例

## ifconfig/ip a

```bash
luxin@shadow:~$ ip a
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN group default qlen 1000
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
    inet 127.0.0.1/8 scope host lo
       valid_lft forever preferred_lft forever
    inet 10.255.255.254/32 brd 10.255.255.254 scope global lo
       valid_lft forever preferred_lft forever
    inet6 ::1/128 scope host
       valid_lft forever preferred_lft forever

    inet 172.17.0.1/16 brd 172.17.255.255 scope global docker0
       valid_lft forever preferred_lft forever
```

### lo回环接口

```bash
1: lo: <LOOPBACK,UP,LOWER_UP>
        inet 127.0.0.1/8 scope host lo
        inet 10.255.255.254/32 scope global lo
```

### 网卡

```bash
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500
        link/ether 00:15:5d:19:a5:ed      ← MAC 地址（物理身份证）
        inet 172.24.126.13/20             ← IP 地址 / 子网掩码
        brd 172.24.127.255                ← 广播地址（系统自动算的）
           scope global eth0
```

> WSL内的Ubuntu网卡设备的编号为eth0,但是大部分现代设备的网卡设备有可能是wlan0

## 不同Linux发行版网卡配置文件

| 发行版        | 配置文件在哪                                                    |
| ---------- | --------------------------------------------------------- |
| Ubuntu 18+ | /etc/netplan/ 目录下 .yaml 文件                                |
| Debian     | /etc/network/interfaces                                   |
| CentOS 7   | /etc/sysconfig/network-scripts/ifcfg-ens33                |
| CentOS 9+  | NetworkManager 管理 /etc/NetworkManager/system-connections/ |

## 方法一 配置DHCP静态IP(Ubuntu/Debian)

### 进入 `/etc/netplan` 目录并查找配置文件（如 `00-installer-config.yaml`)

```bash
cd /etc/netplan
ls
```

### 使用 `sudo` 权限编辑该文件

```bash
sudo nano 00-installer-config.yaml
```

### 修改为如下静态 IP 格式

```yaml
network:
  version: 2
  ethernets:
    eth0: # 请替换为你的实际网卡名称
      dhcp4: no
      addresses:
        - 192.168.1.100/24 # 替换为你的目标静态IP和掩码
      routes:
        - to: default
          via: 192.168.1.1 # 替换为网关IP
      nameservers:
        addresses: # 替换为首选和备用DNS
```

### 应用配置并测试连通性

```bash
sudo netplan apply
```

## 方法二 设置静态IP(Centos/RHEL)

### 进入网络脚本目录并编辑对应的网卡文件

```bash
cd /etc/sysconfig/network-scripts/
sudo vim ifcfg-ens33
```

### 修改以下关键参数

```bash
BOOTPROTO="static"  # 将 dhcp 改为 static
ONBOOT="yes"        # 设置开机自启
IPADDR="192.168.1.100"  # 设定你的静态IP
NETMASK="255.255.255.0" # 设定子网掩码
GATEWAY="192.168.1.1"   # 设定网关
DNS1="8.8.8.8"          # 设定首选 DNS
```

### 重启网络服务使配置生效

```bash
- CentOS 7：`sudo systemctl restart network`
- CentOS 8 / RHEL：`sudo nmcli c reload` 或 `sudo systemctl restart`
```

## 方法三 nmcli设置静态IP

### 查看当前连接名

```bash
nmcli connection show
```

### 设置静态

```bash
nmcli con mod ens33 \
  ipv4.addresses 192.168.1.88/24 \
  ipv4.gateway 192.168.1.1 \
  ipv4.dns "8.8.8.8 114.114.114.114" \
  ipv4.method manual
```

### 生效

```bash
nmcli con up ens33
```

### 修改为DHCP

```bash
nmcli con mod ens33 ipv4.method auto
nmcli con up ens33
```
## 验证静态IP设置

```bash
ip addr
ping 8.8.8.8
```

## 添加第二块网卡

### eth0 管理口（配网关），eth1 业务口（不配网关）

```bash
nmcli con add con-name eth1 type ethernet ifname eth1 \
  ipv4.addresses 10.0.0.10/24 \
  ipv4.method manual
# 注意：不配 gateway

# 但要去特定网段的话，手动加路由
nmcli con mod eth1 +ipv4.routes "10.10.0.0/16 10.0.0.1"
```

### 检查路由表

```bash
ip route
# 正常输出应该只有 1 条 default via
# 内网卡不应该有 default via
```

## 临时添加第二IP

```bash
# 加一个 IP
sudo ip addr add 192.168.1.50/24 dev ens33

# 删一个 IP
sudo ip addr del 192.168.1.50/24 dev ens33

# 清空所有 IP 重新配
sudo ip addr flush dev ens33
sudo ip addr add 192.168.1.88/24 dev ens33
```

## 手动加路由

```bash
# 加一条路由
sudo ip route add 10.0.0.0/8 via 10.0.0.1 dev eth1

# 删一条路由
sudo ip route del 10.0.0.0/8

# 加默认路由（改网关）
sudo ip route add default via 192.168.1.1 dev ens33
```

## 重启网卡

```bash
# 重启单张网卡
sudo ip link set ens33 down
sudo ip link set ens33 up

# 或者用 ifup/ifdown
sudo ifdown ens33 && sudo ifup ens33
```

## 设置添加DNS解析服务器

```bash
# 临时改（重启后丢失）
echo "nameserver 8.8.8.8" > /etc/resolv.conf

# 持久改——看系统
# Ubuntu Netplan：在 yaml 里配 nameservers
# Debian：在 /etc/network/interfaces 里配 dns-nameservers
# CentOS：在 ifcfg 里配 DNS1/DNS2
# 通用：装 resolvconf 或 systemd-resolved
```
