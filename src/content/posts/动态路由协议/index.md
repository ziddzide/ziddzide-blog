# 动态路由协议

## 基础原理

**动态路由：** 网络中的路由器相互通信传递各自的路由信息利用收到的路由信息来更新和维护自己路由表的过程。

**动态路由协议：**

| 单播网络环境中的动态路由协议 | 组播网络环境中的动态路由协议 |
| -------------- | -------------- |
| RIP            | PIM            |
| OSPF           | IGMP           |
| IGRP           | CGMP           |
| EIGRP          | MOSPF          |
| IS-IS          | DVMRP          |
| BGPv4          | CBT            |

## 动态路由特点

```
自动化程度高，减少管理任务，错误率低，但占用网络资源
```

## 协议分类

### 使用范围：

内部网关路由协议（内网中使用）：RIP、OSPF、IGRP、EIGRP、IS-IS
外部网关路由协议（外网中使用）：BGPv4

### 算法分类

a.距离矢量路由协议：

```
距离指有多远（跳数），矢量指从哪个方向（下一跳）。所谓的距离矢量协议就是一个有距离有方向的协议。距离矢量路由协议也被称为听信传闻路由协议。
```

 b.链路状态路由协议：

```
不在听信别人的传闻，而是自己去学习一整张网络拓扑结构环境。每个路由器都主动的把自己知道的路由信息告诉相邻路由器，经过几个更新周期之后，最终每台路由器都能够学到一整张完整的网络拓扑结构。以自己为根节点，计算到达每一个网络节点的最佳路径，不好的路就变成了备份路径。
```

c.混合型路由协议：

```
同时具有距离矢量路由协议和链路状态路由协议的特点。
```

## RIP（距离向量）

### 基础原理

```
RIP是TCP/IP参考模型下被开发出来的第一款动态路由协议，RIP属于标准的距离矢量动态路由协议。
```

| 项目         | RIP 值                 | 说明                 |
| ---------- | --------------------- | ------------------ |
| 协议类型       | 距离向量                  | 不是链路状态（OSPF 才是）    |
| 度量值 Metric | 跳数 Hop Count          | 到目标经过几个路由器         |
| 最大跳数       | 15                    | 16 = 不可达（防止无限循环）   |
| 更新周期       | 30s                   | 每 30s 发一次完整路由表     |
| 超时时间       | 180s（6个周期）            | 收不到就标记为不可达         |
| 抑制时间       | 180s                  | 保持坏路由状态            |
| 端口         | UDP 520               | 传输层用 UDP           |
| 版本         | RIPv1（有类）vs RIPv2（无类） | v2 支持 CIDR/VLSM/认证 |
| 防环机制       | 水平分割、毒性逆转、触发更新、抑制定时器  | 4 种                |
### 工作原理

1. RIP路由协议每隔30s向邻居发送整个路由表信息
2. RIP路由协议以跳数作为度量值，根据跳数的多少来选择最佳路由
3. 最大跳数为15跳，16跳被标记为不可达
4. 经过一系列路由更新，网络中的每个路由器都具有一张完整的路由表的过程，称为收敛

### 防环机制

1. 水平分割: 从哪个接口来的路由不再从哪个接口发出去
2. 毒性逆转: 面对不可发的路由,自动发度量值为16
3. 触发更新: 拓扑一变立刻通知,不等30s Time
4. 抑制定时: 收到坏消息时180s内不再接受新消息

### RIPv1/RIPv2区别

| 特性        | v1               | v2            |
| --------- | ---------------- | ------------- |
| 类别        | 有类（Classful）     | 无类（Classless） |
| 子网掩码      | 不带               | 带             |
| VLSM/CIDR | 不支持              | 支持            |
| 认证        | 无                | 支持明文/MD5      |
| 更新方式      | 广播 255.255.255.5 | 组播 224.0.0.9  |
| 自动汇总      | 强制               | 可选（默认开）       |

### 弱点

1. RIP欺骗: 伪造RIP应答,宣告虚假路由,将流量转发给攻击者机器
2. 无认证(v1): 任何人都可以冒充邻居更新路由
3. 慢收敛 — 拓扑变化后可能长达几分钟才稳定，期间可能有环路

## OSPF(链路协议)

### Router ID

Router ID是一个32比特无符号整数,是路由器在自治系统中的唯一标识.

### Router ID设定

- 通过命令行手动配置，在实际网络部署中，建议手工配置OSPF的Router ID，因为这关系到协议的稳定。
- 通过协议自动选取。
    如果没有手动配置Router ID，设备会从当前接口的IP地址中自动选取一个作为Router ID。其选取顺序是：
    1. 优先从Loopback地址中选择最大的IP地址作为Router ID。
    2. 如果没有配置Loopback接口，则在接口地址中选取最大的IP地址作为Router ID。

> 在路由器运行了OSPF并确定了Router ID后，如果该Router ID对应的接口Down或者接口消失（例如执行了**undo interface loopback** _loopback-number_）或者出现更大的IP地址，OSPF将仍然保持原Router ID。只有重新配置系统的Router ID或者OSPF的Router ID，并且重新启动OSPF进程后，才会进行Router ID的重新选取。

### 和RIP的区别

|      | RIP（距离向量）          | OSPF（链路状态）          |
| ---- | ------------------ | ------------------- |
| 只知道  | 下一跳指向谁 + 多远（跳数）    | 整个网络的拓扑地图           |
| 算法   | Bellman-Ford（赋值更新） | SPF / Dijkstra（自己算） |
| 更新内容 | 完整路由表（30s）         | 只发变化（触发更新）          |
| 度量值  | 跳数（简陋）             | 开销 cost = 100M / 带宽 |
| 收敛速度 | 慢（几百秒）             | 快（几秒）               |
| 环路风险 | 高（距离向量通病）          | 天然无环（SPF 树状结构）      |
| 区域划分 | 不支持                | 支持区域 Area，分而治之      |

### 核心流程

#### 第1阶段：发现邻居（Hello）

```
每 10s 发 Hello 包到 224.0.0.5
收到回应 → 建立邻居关系
```

#### 第2阶段：同步数据库（LSDB）

```
邻居之间交换 LSA（链路状态通告）
每个路由器都有一张完全相同的 LSDB全网拓扑图
```


#### 第3阶段：算路（SPF / Dijkstra）

```
用 Dijkstra 算法算最短路径树,把最短路径放进路由表,自己算的，没有环路
```

#### 第4阶段：维护（只发变化）

```
链路断了 → 马上发一条新的 LSA
其他路由器收到 → 重新跑 SPF
收敛时间 ≈ 几秒（RIP 要几百秒）
```

### 邻居状态

```
    Down ──→ Init ──→ 2-Way ──→ ExStart ──→ Exchange ──→ Loading ──→ Full
                           │
                           ↓
                         DR/BDR 选举在这里完成
                         2-Way 就停（非DR）或继续走（DR）
```

| 状态       | 发生了什么                         | 记住方式    |
| -------- | ----------------------------- | ------- |
| Down     | 初始                            | 无任何情况   |
| Init     | 收到对方的 Hello，但还没跟对方打招呼         | 看到邻居    |
| 2-Way    | 双方都收到了对方的 Hello               | 握手      |
| ExStart  | 选主从（Master/Slave），决定谁先发       | 选主从     |
| Exchange | 交换 DBD 摘要（只发目录，不发全文）          | 交换LSA信息 |
| Loading  | 发现对方有的自己缺 → 发 LSR 请求 → 收到 LSU | 补齐信息    |
| Full     | LSDB 完全同步                     | 齐了      |

#### 补充:广播网络关键机制-DR/BDR

##### DR/BDR（广播网络中的关键机制）

    问题：如果 10 台路由器在一个网段，两两 Full 要建 45 个邻接
          每台路由器发 9 份 LSA

    解决：选一个 DR、一个 BDR
          所有非DR只跟 DR/BDR 同步
          非DR之间停在 2-Way，不建 Full

    结果：10 台路由器 → 只要建 17 个邻接（DR跟9人+BDR跟9人-1重）
          效率翻倍
      
##### 角色

| 角色      | 谁选            | 谁跟他 Full                  |
| ------- | ------------- | ------------------------- |
| DR      | Hello 里优先级最高的 | 所有人跟他 Full                |
| BDR     | 优先级第二高的       | 所有人跟他 Full                |
| DROther | 剩下的           | 只跟 DR/BDR Full，互相停在 2-Way |

### LSA 类型

| 类型     | 名字           | 谁发的   | 内容              | 作用范围   |
| ------ | ------------ | ----- | --------------- | ------ |
| Type 1 | Router LSA   | 每台路由器 | 自己的直连接口+开销      | 本区域    |
| Type 2 | Network LSA  | DR    | 本网段所有路由器列表      | 本区域    |
| Type 3 | Summary LSA  | ABR   | 把本区域路由汇总后告诉别的区域 | 跨区域    |
| Type 4 | ASBR Summary | ABR   | 告诉其他区域 ASBR 在哪  | 跨区域    |
| Type 5 | External LSA | ASBR  | 引入的外部路由（如静态重分布） | 整个 AS  |
| Type 7 | NSSA LSA     | ASBR  | 特殊区域里用的外部路由     | NSSA 内 |

### 区域设计

```
             骨干区域 Area 0
      ┌──────────┬──────────┐
      │          │          │
    Area 1    Area 2    Area 3


    - 所有非骨干区域必须直连 Area 0
    - Area 0 是枢纽，跨区域路由必须经过 Area 0
    - ABR（区域边界路由器）：连接 Area 0 和其他区域
    - ASBR（自治系统边界路由器）：把外部路由引入 OSPF
```


## 额外的内容

### Windows主机和企业路由器

```
Windows主机以及Linux主机只有路由表没有动态路由协议,在家用主机或者办公主机拥有动态路由的情况下会对环境造成大量的路由计算难度,因此终端设备只有路由表,没有动态路由协议.
```

### 家用路由器和企业路由器的区别

```
家用路由器一般没有动态路由协议,因为出网点只有一个上层设备,且路由器隔离的内网环境只有简单设备,不需要动态路由计算路由表
```
